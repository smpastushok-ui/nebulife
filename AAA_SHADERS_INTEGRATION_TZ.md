# ТЗ: Інтеграція ААА+ шейдерів (Exosphere та System View)

## 1. Мета та Контекст
Гравець щойно побачив "Living Orrery" (наш новий лендінг) з високодеталізованим Сонцем, хмарами на Землі та кільцями Юпітера, і хоче бачити таку ж якість у самій грі.

Треба інтегрувати ці круті шейдери з `LivingOrreryHero.tsx` у два основних рівня гри:
1. **Рівень "Екзосфери" (`PlanetGlobeView.tsx`)**: 3D огляд однієї планети. Тут потрібна максимальна деталізація планети (AAA+ шейдери).
2. **Рівень "System View" (`SystemPlanet3DLayer.ts`)**: Огляд всієї системи. Тут потрібно замінити тільки Зорю (Сонце) на наше живе пульсуюче Сонце. Планети на цьому рівні залишаються простими (спрайти або базові меші), щоб не спалити GPU.

## 2. Завдання (To-Do List)

### 2.1. Екзосфера: AAA+ Планети (`PlanetGlobeView.tsx`)
1. Експортувати `planetVertexShader` та `planetFragmentShader` з `LivingOrreryHero.tsx` у новий файл (напр., `packages/client/src/game/shaders/PlanetShaders.ts`).
2. В `PlanetGlobeView.tsx` знайти створення 3D-моделі планети (зараз там використовуються текстури або прості шейдери `rockySurfaceFrag`, `gasGiantFrag`).
3. Замінити існуючі матеріали планети на єдиний новий `THREE.ShaderMaterial` з нашими крутими шейдерами.
4. Налаштувати мапінг:
   - `planet.type` з ігрового ядра (`gas-giant`, `ice-giant`, `rocky`, `terrestrial`, `dwarf`) переводити у відповідний цілочисельний `planetType` для шейдера (0 = Землеподібна, 1 = Юпітер, 2 = Марс і т.д.).
   - Передавати `baseColor` та `altColor` в шейдер на основі біому/seed-у планети.
5. Забезпечити підтримку кілець (як у `LivingOrreryHero.tsx`, з використанням `THREE.Points`) для `gas-giant` та `ice-giant`, якщо у них є кільця (`hasVisibleRingSystem`).
6. **Оптимізація**: Шейдер важкий (використовує FBM). В залежності від `getDeviceTier()`, можна зменшувати кількість ітерацій шуму (октав).

### 2.2. System View: AAA+ Зоря (`SystemPlanet3DLayer.ts`)
1. Експортувати `sunVertexShader`, `sunFragmentShader`, а також `corona` шейдери з `LivingOrreryHero.tsx` у новий файл (напр., `packages/client/src/game/shaders/SunShaders.ts`).
2. У `SystemPlanet3DLayer.ts` зараз для зорі створюється `THREE.Mesh` з `THREE.MeshStandardMaterial` (`this.starSphere = new THREE.Mesh(...)`).
3. Замінити цей матеріал на наш `THREE.ShaderMaterial` з ефектом плазми.
4. Налаштувати кольори (`colorDark`, `colorBase`, `colorBright`, `colorCore`) в залежності від спектрального класу зорі (`spectralClass` O, B, A, F, G, K, M). Наприклад, для Блакитних Гігантів використовувати сині/білі відтінки, для Червоних Карликів — темно-червоні.
5. Додати ефект корони (`AdditiveBlending`, `BackSide`), як у `LivingOrreryHero`.

## 3. Очікуваний результат
1. В Екзосфері (на екрані планети) гравець бачить надзвичайно деталізовану процедурну планету з хмарами, океанами (якщо є) або бурями (для газових гігантів).
2. В System View (на екрані системи) гравець бачить пульсуючу зорю відповідного спектрального класу, що освітлює простіші моделі планет. Продуктивність (FPS) не страждає, оскільки важкі планетні шейдери не рендеряться для всієї системи одночасно.
