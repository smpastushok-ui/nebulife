## 1. Що це і чим відрізняється від поточної Академії

### Поточна Академія (залишається):
- **Урок дня** — щоденний короткий урок, що генерується Gemini AI
- **Квест** — short-form quizzes з нагородою
- **Тести** — швидкі вікторини
- **Новини** — Weekly Digest

Це залишаємо без змін. Цикл daily engagement не чіпаємо.

### Нова частина — **«Бібліотека знань»** (Library):
- Окрема вкладка `Бібліотека` поруч з `Урок / Квест / Тести / Новини / Профіль`
- Це довідник, не курс. Гравець заходить коли хоче дізнатися щось конкретне
- Структуровано як вікіпедія: розділи → теми → уроки
- Кожен урок — добре оформлена сторінка з текстом, ілюстраціями, схемами, аудіо
- Уроки **не AI-generated** — вони написані експертами (sonnet-агентами під суворим брифом
  з посиланнями на першоджерела, з фактчекінгом і бібліографією)
- Гравець може читати, слухати озвучку, ділитися посиланнями, відмічати прочитане

### 2.2. Чому MDX, а не БД / CMS / Markdown

| Підхід | Плюси | Мінуси |
|---|---|---|
| **MDX в репо** ✓ | Version-controlled, code-review через PR, доступне для AI-агентів, тримаємо все в Git | Build-step потрібен |
| Markdown в БД (Neon) | Live editing | Кожен запит — DB hit, складніше ревью, важко версіонувати |
| Headless CMS (Sanity/Contentful) | UI для редакторів | Зайвий сервіс, $ за seats, vendor lock-in |
| HTML файли | Простіше всього | Немає типізації фронтматтера, складно вставляти React-компоненти |

**MDX = Markdown + React-компоненти**. Дозволяє писати:

```mdx
---
slug: black-holes-intro
section: astrophysics
title: Чорні діри — вступ
difficulty: beginner
readingTime: 12
heroImage: /academy/images/black-holes-intro/hero.webp
audioUrl: /academy/audio/black-holes-intro.uk.mp3
sources:
  - { title: "Event Horizon Telescope Collaboration, ApJ 2022", url: "..." }
  - { title: "Hawking Radiation, Nature 1974", url: "..." }
lastVerified: 2026-05-01
---

# Чорні діри

Зорі живуть мільярди років...

<HeroImage src="..." alt="..." caption="..." />

<Diagram type="schwarzschild-radius" />

<AudioPlayer src="..." />

<Glossary terms={['event-horizon', 'singularity']} />

<FurtherReading />
```

### 2.3. Розмір додатку — як НЕ роздути

**Стратегія lazy-load + CDN:**

1. **Бандл застосунку** — не зростає взагалі. Бібліотека вантажиться через `React.lazy`.
2. **Індекс бібліотеки** (~50KB на мову) — вантажиться при першому відкритті, кешується в IndexedDB.
3. **Текст уроку** (~5-15KB MDX) — fetch-on-demand з `/academy/content/{section}/{slug}.{lang}.html`
   (MDX компілюється у статичний HTML на build-time).
4. **Зображення** — на Vercel CDN, `loading="lazy"`, формат webp/avif.
5. **Аудіо** — MP3 на Vercel Blob або в `/public/academy/audio/`, вантажиться при кліку
   на `<AudioPlayer>` (стрімінгом, не цілим файлом).

**Розрахунок розмірів:**

| Тип | Кількість | Розмір 1 | Сумарно |
|---|---|---|---|
| MDX уроки (uk + en) | 90 × 2 = 180 | ~8KB | 1.4 MB |
| Зображення (3-5 на урок) | ~400 | ~80KB webp | 32 MB |
| Аудіо (uk + en) | 180 | ~3MB MP3 | 540 MB |
| **Total на CDN** | | | ~575 MB |
| **Бандл застосунку** | | | **+0 KB** |

CDN-вартість на Vercel Pro — $0 для 1 TB bandwidth/міс. Поки трафік не пройде 1 TB, безкоштовно.

### 2.4. Auto-генерація ілюстрацій (build-time)

В MDX-файлі автор пише placeholder:

```mdx
<GenImage
  prompt="Schematic diagram of Schwarzschild radius around a black hole, scientific illustration style, dark background, monospace labels"
  alt="Радіус Шварцшильда — границя без повернення"
  caption="Радіус, на який стискається маса при колапсі в чорну діру"
  width={800}
  height={500}
  cacheKey="black-holes-intro-schwarzschild"
/>
```

**Build script** (`scripts/academy-build.ts`):

1. Парсить всі MDX
2. Знаходить `<GenImage>` теги
3. Для кожного `cacheKey` перевіряє чи вже є файл в `/public/academy/images/{cacheKey}.webp`
4. Якщо немає — викликає Gemini Imagen / Imagen-4 / Nano Banana з промптом
5. Зберігає у webp 800px wide
6. У compiled HTML заміняє `<GenImage>` на `<img src="/academy/images/{cacheKey}.webp">`

Це детерміновано — один і той самий `cacheKey` дає один файл, регенерація тільки якщо
видалити кеш-файл або явно передати `forceRegenerate: true`.

### 2.5. Озвучка (TTS)

В фронтматтері кожного уроку є `audioUrl`. Build script:

1. Бере plain-text версію уроку (MDX → markdown → strip-tags)
2. Викликає **Google Cloud TTS** (голоси `uk-UA-Wavenet-A` для жінки, `uk-UA-Wavenet-B` для чоловіка)
   або **OpenAI TTS** (`tts-1-hd`, голос `nova` для жінки, `onyx` для чоловіка)
3. Зберігає MP3 в `/public/academy/audio/{slug}.uk.mp3` і `{slug}.en.mp3`
4. Frontmatter автоматично оновлюється з `audioUrl`

Гравець бачить компонент `<AudioPlayer>` зверху уроку:
```
[▶] Послухати урок (12 хв) | Голос: A.S.T.R.A. ▾ | 1.0× ▾
```

**Чому pre-generated, а не runtime TTS:** дорого, latency, залежність від API. 90 уроків × 12 хв
× 2 мови = ~36 годин аудіо, ~$50 одноразово через Google TTS, потім безкоштовно з CDN.

### 2.6. Інтерактивні елементи (поза текстом)

Деякі уроки потребують більше ніж зображення. Список готових React-компонентів:

| Компонент | Призначення |
|---|---|
| `<Timeline events={[...]} />` | Хронологія (місії, відкриття) |
| `<ScaleComparison items={[...]} />` | Інтерактивне порівняння розмірів (Земля vs Юпітер vs Сонце vs Бетельгейзе) |
| `<OrbitDiagram bodies={[...]} />` | Анімована схема орбіт (PixiJS) |
| `<SpectrumViewer star="..." />` | Спектр зорі з підсвіченими лініями поглинання |
| `<HRDiagram />` | Інтерактивна діаграма Герцшпрунга-Рассела |
| `<RedshiftSlider />` | Слайдер червоного зміщення з демонстрацією ефекту |
| `<GravityWell mass={...} />` | Викривлення простору-часу під масою |
| `<MissionMap year={...} />` | Карта активних місій на дату |
| `<Quiz questions={[...]} />` | Вбудована вікторина в кінці уроку |
| `<Glossary terms={[...]} />` | Глосарій термінів з підказками |
| `<FurtherReading items={[...]} />` | Список першоджерел |

Компоненти живуть у `packages/client/src/academy-library/components/` і завантажуються тільки
коли потрібні (dynamic import).

### 5.1. Брифінг для кожного агента

```
ROLE: Експертний автор-популяризатор з астрофізики/космонавтики (PhD-рівня)
SECTION: Астрономія (Section 1)
TOPICS: 10 уроків з detailed list
LANGUAGES: uk + en (паралельно)
LENGTH: 1500-2500 слів кожен урок
TONE: Дорослому читачу, без снобізму, без емодзі. Складне — простими словами.
ACCURACY: Кожна цифра/дата/факт має бути перевірена. На сумнівні твердження — посилання.
SOURCES: Кожен урок має 5-10 first-class sources (NASA/ESA/arXiv/Nature/AAS journals)
LAST_VERIFIED: 2026-05-03
FORMAT: MDX файли в packages/client/src/academy-library/content/{section}/{slug}.{lang}.mdx
PLACEHOLDERS:
  - <GenImage prompt="..." cacheKey="..." /> — там де потрібна ілюстрація
  - <Diagram type="..." /> — для схем
  - <Quiz questions={[...]} /> — наприкінці уроку
  - <Glossary terms={[...]} /> — список термінів
  - <FurtherReading items={[...]} /> — посилання на джерела
DELIVERABLE: 10 уроків × 2 мови = 20 MDX файлів
RUNTIME: ~30-45 хв на агент (включно з фактчекінгом)
```
